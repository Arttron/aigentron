# Skill: i18n (Internationalization)
**Applies to:** Architect, Backend, Frontend, Designer, Reviewer

---

## Stack

- `next-intl` for Next.js App Router
- `nestjs-i18n` for the NestJS backend
- Locale files: one JSON per language

## Supported locales

```typescript
export const locales = ['en', 'ru'] as const;
export type Locale = typeof locales[number];
export const defaultLocale: Locale = 'en';
```

## Next.js setup (next-intl)

```
app/
├── [locale]/
│   ├── layout.tsx
│   ├── page.tsx
│   └── dashboard/page.tsx
└── i18n.ts

messages/
├── en.json
└── ru.json
```

```typescript
// middleware.ts
import createMiddleware from 'next-intl/middleware';
import { locales, defaultLocale } from './i18n.config';

export default createMiddleware({ locales, defaultLocale, localePrefix: 'always' });
export const config = { matcher: ['/((?!api|_next|_vercel|.*\\..*).*)'] };
```

```typescript
// Server Component
import { useTranslations } from 'next-intl';
export default function Page() {
  const t = useTranslations('Dashboard');
  return <h1>{t('title')}</h1>;
}
```

## NestJS backend (nestjs-i18n)

```typescript
I18nModule.forRoot({
  fallbackLanguage: 'en',
  loaderOptions: { path: path.join(__dirname, '/i18n/'), watch: true },
  resolvers: [{ use: QueryResolver, options: ['lang'] }, AcceptLanguageResolver],
}),
```

## Rules

### Coder (backend/frontend)
- Never hardcode user-facing strings — always use translation keys.
- Add keys to every locale file when creating new UI.
- Use descriptive nested keys: `Feature.action.state`.

### Designer
- All visible text via `useTranslations()`, never raw strings.
- RTL-safe layout: CSS logical properties (`margin-inline-start`, `inset-inline-end`) instead of
  physical `left`/`right`.

### Reviewer
- No hardcoded user-facing strings in components.
- Every locale file updated when new keys are added.
- Locale parameter passed through API calls where needed.

## Layout safety across languages

Different languages break layouts predictably — translated text can run 30-40% longer than
English. Test every component against every configured locale.

| Problem | Cause |
|---------|-------|
| Text overflow | Longer translated strings |
| Broken buttons | Fixed width + longer text |
| Wrapped navigation | Nav items longer in some languages |
| Icon/text misalignment | Text wraps but icon stays fixed |

Size text containers with `min-width` + padding (never a fixed `width`), let nav wrap, and let
table headers wrap by content — in the component's CSS Module:

```css
/* component.module.css */
/* ❌ WRONG: .save { width: 128px } — clips longer translations */
.save { min-width: 80px; padding: 0 16px; }             /* ✅ grows with the text */
.nav  { display: flex; flex-wrap: wrap; gap: 16px; }    /* ✅ wraps, not overflow:hidden */
.th   { min-width: 100px; max-width: 200px; padding: 8px 16px; overflow-wrap: break-word; }
```
```tsx
<button className={styles.save}>{t('actions.save')}</button>
<nav className={styles.nav}>…</nav>
<th className={styles.th}>{t('table.columnName')}</th>
```

## Screenshot checklist across locales

```bash
npm run preview -- --url=/en/dashboard
npm run preview -- --url=/ru/dashboard
npm run preview -- --url=/en/dashboard --mobile
npm run preview -- --url=/ru/dashboard --mobile
```

Check: button sizes consistent, navigation fits or wraps gracefully, no clipped text, cards same
height across locales, no unwanted horizontal scroll, icons stay aligned, dates/numbers formatted
per locale.
